# 实验结果分析指南

本文档旨在帮助你理解 `outputs/` 目录下生成的实验图表，以及如何评估模长解耦（Norm Decoupling）的效果。

## 1. 实验图表解读

### 图表 1: 采样轨迹图 (Sampling Trajectories)
*   **文件名**: `traj_baseline.png` / `traj_decoupled.png`
*   **内容**: 展示了从 $t=T$ (纯噪声) 到 $t=0$ (生成数据) 的采样过程。
*   **图例**:
    *   **红点 (Start)**: 初始的高斯噪声点 $x_T$。
    *   **蓝点 (End)**: 最终生成的样本点 $x_0$。
    *   **线条**: 采样过程的移动轨迹。颜色深浅通常代表时间进程（浅色为早期，深色为晚期）。
*   **如何分析**:
    *   **理想情况**: 轨迹应该像河流汇入大海一样，从无序的噪声区域迅速汇聚到螺旋线流形上。
    *   **直线性**: Decoupled 模型如果方向预测准确，其轨迹通常比 Baseline 更直，因为并没有被错误的模长干扰方向。
    *   **收敛性**: 观察蓝点是否清晰地构成了螺旋线的形状。如果是一团模糊的云，说明没有训练好。

### 图表 2: 模长热力图 (Norm Heatmap) —— **核心指标**
*   **文件名**: `norm_baseline_t{t}.png` / `norm_decoupled_t{t}.png`
*   **内容**: 在 2D 平面上网格化采样点 $(x, y)$，计算模型在时刻 $t$ 预测的噪声模长 $\|\epsilon_\theta(x, y, t)\|$。
*   **坐标轴**: X, Y 坐标，范围通常是 $[-2, 2]$。
*   **颜色含义 (Colorbar)**: 表示预测噪声向量的长度（模长）。
    *   **亮色 (黄/红)**: 模长较大。意味着模型认为该点距离目标流形较远，或者噪声水平很高。
    *   **暗色 (蓝/紫)**: 模长较小。
*   **如何分析 (实验验证的关键点)**:
    *   **Baseline Model**: 你会发现在螺旋线的**空隙 (Voids)** 区域（即两条臂之间），模长往往会**坍缩到接近 0**（变成深蓝色/紫色）。
        *   *原因*: 标准 MSE Loss 导致模型在不确定方向时，倾向于输出所有可能方向的平均值（即 0 向量）。
    *   **Decoupled Model**: 在空隙区域，你应该能看到**更均匀或连续的模长分布**（通常接近 $\sqrt{2} \approx 1.414$，因为 2D 高斯噪声的期望模长）。
        *   *结论*: 这证明了解耦模型成功分离了“方向的不确定性”和“模长的确定性”。即使不知道往哪走，它也知道步子该迈多大。

### 图表 3: 向量场图 (Vector Field)
*   **文件名**: `vec_baseline_t{t}.png` / `vec_decoupled_t{t}.png`
*   **内容**: 展示了去噪方向 ($- \epsilon_\theta$)。
*   **箭头**:
    *   **方向**: 指向模型认为的流形位置。
    *   **长度**: 通常归一化处理过，只看方向流向。
*   **如何分析**:
    *   查看在远离螺旋线的地方，箭头是否能正确指向最近的螺旋臂。
    *   对于 Decoupled 模型，检查其向量场是否比 Baseline 更平滑、一致性更强。

## 2. 为什么螺旋线拟合不好？(Troubleshooting)

如果你发现生成的螺旋线是断裂的、模糊的，或者完全就是一团散沙，通常有以下几个原因：

### 原因 A: 模型缺乏高频拟合能力 (High Frequency Bias)
**这是最常见的原因**。普通的 MLP（多层感知机）极其不擅长拟合低维坐标输入的高频函数（例如紧密的螺旋结构）。这被称为 "Spectral Bias"。
*   **解决方案**: 引入 **Gaussian Fourier Features (高斯傅里叶特征)**。将输入 $(x, y)$ 映射到高维对角空间 $[\sin(2\pi x), \cos(2\pi x), ...]$。这能极大提升 MLP 对细节的捕捉能力。**（我将在接下来的代码修改中为你实现这一点）**

### 原因 B: 训练步数与容量不足
*   **Epochs**: 200 epochs 对于 Toy Experiment 有时太少，建议增加到 **1000** 或 **2000**。因为数据量小，训练很快。
*   **Timesteps**: 200 步扩散可能导致每一步除去的噪声太多，误差累积大。建议增加到 **500** 或 **1000**。

### 原因 C: 学习率调度
*   如果 Loss 下降很快但测试效果差，可能是过拟合或学习率降得太快。
